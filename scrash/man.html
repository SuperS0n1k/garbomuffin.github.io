NAME scrash - a SCRAtch SHell, based on bash DESCRIPTION Scrash is a recreation of many of the core features of bash, in Scratch. [1] It is intended to be more of a "demo" than a finished product. [2] SYNTAX This section is unfinished. The syntax is really just a heavily stripped down version of bash's syntax. Specifically, it supports: comments quoting escaping variables directories (/, ~, ., ..) semicolons to do multiple commands on one line COMMENTS A # defined a comment. Everything passed that # will be effectively ignored by the parser. echo abc #def abc VARIABLES A variable definition is in the form: my_variable_name="Hello, world!".
You should use quotes to define multiword variables.
The name of the variable must be a single word. A variable reference is in the form: $my_variable_name.
Variable names are not case sensitive. my_variable_name="Hello, world!" echo $my_variable_name Hello, world! DIRECTORIES Special characters: ~ Refers to the home directory. (/home/user/) . Refers to your current directory. .. Refers to the directory up one level of your current directory. (/home/user/ => /home/) AUTHORS Written by @GarboMuffin Text engine by @TriforceMayhem Some hacked blocks by @griffpatch COMMANDS The following is a list of most of the commands, what they do, and an example how of to use them.
Some commands that are either "hidden" (parsedir) or very self explanatory (date) are not listed. echo "Echos" back the given arguments. echo abc def abc def history Lists your command history. history    1   echo abc def    2   history ... clear Clears the terminal. clear ls Lists the contents of a directory. ls a.sh  b.sh cat Lists the contents of file(s). cat a.sh echo This is in a.sh ./b.sh rm Remove a file or directory. rm a.sh pwd Prints your current working directory. pwd /home/user/ ed A list based text editor. ed a.sh alias List or define aliases. alias alias edit='ed' alias editor='ed' ... alias a=b cd Change the current working directory. Refer to directory syntax for more details. cd .. mkdir Makes directories. mkdir a export Exports your current session and generates an "export code". export import Replaces your current session with a given "export code". import BUGS A list of bugs can be found here. [1] More specifically: The easier features of bash. [2] I've always complained about Scratch "OS"es not having an editable file system or text editor. A large part of the motivation for this was for me to prove that dynamic file systems are a) possible, b) not as complicated as they may seem [3]. [3] The implementation this uses is rather poor in quality and probably will find a way to break on its own. This is by no means a "reference implementation".
